# ADRs (Architecture Decision Records)

ADRs live in this folder.

Write an ADR when a decision has long-term architectural impact. ADRs should be short, dated, and stable. If the decision changes, write a new ADR.

### Naming
Use a numeric prefix and short slug:

- `0001-multi-tenancy-model.md`

### Template
Copy `TEMPLATE.md` to start a new ADR.
